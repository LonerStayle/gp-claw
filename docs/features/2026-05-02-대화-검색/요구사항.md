# 요구사항: 대화 검색

## 1. 배경/목적
- 방이 많아지면서 과거에 주고받은 내용·코드·결정이 어디 있는지 찾기 어려움.
- 키워드와 조건(날짜/방/role)으로 메시지 단위 검색 → 결과를 클릭하면 해당 방의 해당 메시지로 점프하여 즉시 컨텍스트를 회수.
- 의미 기반 검색 및 파일 내용 검색은 본 PRD에서 분리(추후 별도 PRD).

## 2. 사용자 스토리 / 시나리오
- **스토리 A**: 사용자로서, 상단 검색바에 키워드를 입력하면 모든 방의 메시지에서 일치 결과를 즉시 보고, 그 중 하나를 클릭하면 해당 방으로 이동해 메시지가 하이라이트된 상태로 보고 싶다 — 이전 대화 맥락을 빠르게 회수하기 위해.
- **스토리 B**: 사용자로서, 사이드바 방 목록에서 방 이름으로 필터링하고 싶다 — 방이 많아질 때 빠르게 원하는 방을 열기 위해.
- **스토리 C**: 사용자로서, 날짜 범위·방·role(user/assistant/tool/system)을 조합해 결과를 좁히고 싶다 — 검색 범위를 넓게 시작해 점진적으로 좁혀가기 위해.

## 3. 기능 요구사항 (FR)
- **FR-1**: 전역 검색바 — 모든 방의 메시지를 키워드로 검색 (대소문자 무시 + 부분 일치).
- **FR-2**: 검색 대상 — user / assistant / tool / system 4가지 role 메시지 모두 포함.
- **FR-3**: 결과 항목 표시 — 메시지 스니펫(키워드 위치 기준 앞뒤 컨텍스트) + 방 이름 + 작성 시각 + role 표시.
- **FR-4**: 결과 클릭 동작 — 해당 방으로 자동 이동 + 해당 메시지로 자동 스크롤 + 키워드 하이라이트.
- **FR-5**: 결과 정렬 — 메시지 작성 시간 내림차순(최신순) 기본.
- **FR-6**: 페이지네이션/무한스크롤 — 1회 50개 로드, 추가 로드 가능.
- **FR-7**: 검색 결과 화면의 필터 — 날짜 범위(오늘 / 최근 7일 / 최근 30일 / 사용자 지정), 방(단일 또는 다중 선택), role(user/assistant/tool/system 체크박스).
- **FR-8**: 사이드바 방 목록 필터 — 방 이름 부분일치(대소문자 무시).
- **FR-9**: 결과 0건 처리 — "결과 없음" 안내를 명시적으로 표시.

## 4. 비기능 요구사항 (NFR)
- **NFR-1 (성능)**: 정량 SLA는 정의하지 않는다. 체감상 즉시성 있는 응답을 목표로 하되, 측정 기준은 본 PRD 범위 밖.
- **NFR-2 (보안/접근제어)**: 단일 사용자 시스템이므로 별도의 권한·필터링 없이 모든 방의 모든 메시지가 검색 대상.
- **NFR-3 (호환성)**: 기존 메시지 데이터 모델은 변경하지 않는다. 검색 성능을 위한 신규 인덱스 추가는 허용. 신규 외부 인프라(임베딩 서비스/벡터 DB 등) 도입 금지.

## 5. 범위 밖 (Out of Scope)
- 의미 기반 검색 (임베딩 / 벡터스토어) — 별도 PRD로 분리.
- 첨부 파일 내용(PDF/HWP/오피스 등) 자체 검색 — 파일 첨부 기능 안정화 후 별도 결정.
- 정규식 / AND·OR·NOT 등 고급 쿼리 문법.
- 검색 쿼리 저장(즐겨찾기) 및 결과 익스포트.
- 검색 로그·감사 레코드.

## 6. 수용 기준 (Acceptance Criteria)
- **AC-1**: 상단 검색바에 키워드를 입력하면 결과 리스트가 최신순으로 50개 단위 표시된다. (Yes/No)
- **AC-2**: 결과 항목 클릭 시 해당 방으로 이동하고, 해당 메시지로 자동 스크롤되며, 키워드가 하이라이트된다. (Yes/No)
- **AC-3**: 사이드바 방 목록에서 방 이름으로 필터링이 가능하다(부분일치, 대소문자 무시). (Yes/No)
- **AC-4**: 날짜(오늘/7일/30일/사용자 지정) + 방 + role 필터를 조합 적용할 수 있다. (Yes/No)
- **AC-5**: 검색은 대소문자를 무시하며 부분 일치를 지원한다. (Yes/No)
- **AC-6**: 결과가 0건일 때 "결과 없음" 메시지가 명시적으로 표시된다. (Yes/No)
- **AC-7**: 결과가 50개를 초과할 때 추가 로드(더 보기 또는 무한스크롤)가 가능하다. (Yes/No)
